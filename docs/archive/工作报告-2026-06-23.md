## 四、本阶段揪出并修掉的 2 个真 bug

1. **账户名取值用错方法**（`b249092`）
   `probeAccount` 读 `body.innerText`，但账户名"聚量TS-捷沅3"藏在**折叠的账户切换器=隐藏元素**里，innerText 取不到 → 误判账户漂移。改读 `outerHTML` 即通。
   *（附带澄清：此前怀疑的"C 盘满导致冷启动挂死"是误判——C 盘 0.1G 下页面正常加载，真因就是这个取值方法。）*

2. **jie3 的握手 advId 填错**（`3c2e533`）
   `jie3.advId` 原本填成**捷沅6 的 id**，jie3 靠"暖 profile + URL 参数"硬切到捷沅3——**会话一被重置就失灵**（握手落捷沅6、再切捷沅3 被弹回）。改成捷沅3 自己的 id → 握手直落捷沅3，**冷会话也能收敛**。这是暖 profile 长期遮住的潜伏 bug，被 jie6 实验里的会话重置揪出来，等于提前排了个雷。

## 五、★ 已知未修 bug 清单（按"是否影响 jie3"排序）

### A. 直接关系 jie3 日常（优先）

- **#1 `delete jie3` 报"将删 0"存疑** —— sync 说有 13 个待重传，delete 却算 0 个要删。**很可能是对的**（被拒副本在历史轮次已删过），但**没拿平台数据交叉核实**。若实为匹配漏，jie3 的删除腾槽会失效。→ **建议尽快核实。**
- **#2 登录探测坏** —— 冷/登出的 profile 上 `login` 步不触发（agent 落地页 & redirect/ad 在检查点都不亮 `/login`）。**只在 jie3 profile 登出时才犯**；现在它一直登录着所以不发作，但这是 jie3 鲁棒性最大短板（cookie 过期就卡）。

### B. 没 live 验证过、可能藏 bug

- **#3 `login()` 表单登录逻辑** —— 从没真跑（jie3 一直登录态），选择器可能漂移。
- **#4 `monitor`/遥测录制段** —— 只验了统计，录制本身没 live 测。
- **#5 `guard` 的"掉登录自愈"成功路径** —— 依赖 `login()`，等于也没真验（验过的是"拒绝错账户操作"的安全路径）。

> #2/#3/#5 其实是**同一个根**：登录探测+登录这条链没打通。修它一箭三雕。

## 七、当前可用性结论

> **"jie3 测试通道 + 非上传操作 + profile 保持登录"这个范围内，CLI 能稳定用。**

但它**还不是"换台机器、彻底冷启动都能自愈"的鲁棒**——登录恢复是最大短板，下次 profile 登出就会暴露。"依赖本机环境"主要就是依赖那个**暖登录态的 profile**，外加常规配置（Chrome/ffmpeg 路径、台账盘 H:、输出盘 I: 空间、git 代理）。

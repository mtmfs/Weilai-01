# 恢复手册（RECOVERY）—— 退出码对照

每条命令结束都有退出码；非 0 时按下表处理。**绝大多数情况：先重跑同一条命令。**

| 码 | 名 | 含义 | 你该做什么 |
|---|---|---|---|
| 0 | OK | 成功 | — |
| 1 | RUNTIME | 未归类运行时错 | 重跑一次；持续则贴日志给维护者 |
| 2 | E_USAGE | 命令行参数错（含中文、未知命令等） | 只用 ASCII 名（jie3/jie6）；中文写进 targets/*.json |
| 10 | E_DRIFT | 账户漂移未恢复 | 用双实例（jie3→9222 / jie6→9223 各自 profile）；重跑 `ready` |
| 11 | E_LOGIN | 掉登录、自动重登失败 | 按提示登录母账号后重跑（终端用 `! <命令>`） |
| 12 | E_SIG | 签名 / SSO 会话过期 | 重跑 `ready`（会重握手）后再继续 |
| 13 | E_ROI | ROI 超上限锁住增删 | 去计划编辑页把净成交 ROI 降到上限内保存，再重跑 |
| 14 | E_SELECTOR | 平台改版、选择器漂移 | **停** —— 需要维护者（AI）修选择器，别硬试 |
| 15 | E_GESTURE | 上传拖拽框未弹出文件框 | 重跑 `ready` 后再传；持续则联系维护者 |
| 16 | E_FREEZE_SKIP | 个别批次冻结、已跳过 | 非致命；下一轮 `status` 后补传剩余即可 |
| 20 | E_CONFIG | 配置缺失 / 格式错 | 检查 `system.json` 与 `targets/*.json` |

## 常见场景
- **「AI 一键修复」弹窗盖住界面** → `guard` 会自动关掉再操作，无需手动。
- **上传卡在某个文件很久** → 逐文件超时会把它踢回队列、先提交已完成的，不会再陪等。
- **跑一半被顶号漂到捷沅6** → 命令报 `E_DRIFT` 并停；用双实例可从根上避免。

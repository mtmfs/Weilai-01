## 2. 本轮变更明细（changelog）

### ✅ 重点①(A4)：通道事实归一到 `channels/*.json`，消除 maxUploads 漂移

**现象**：`lib/state.mjs::DEFAULT_CHANNELS` 把 `account/aavid/planId/maxUploads` 等通道事实又硬编一份，与 `channels/*.json` 重复；且 `jie3.maxUploads` 已漂移——`state.mjs`=**5**，`channels/jie3.json` 与线上台账=**7**。

**为何潜伏未爆**：`ledger.mjs::applySync` 与 `sync.mjs` 在**每次提交/读取**时 `loadChannels()(读 channels) → syncChannels` 覆盖台账 channels，磁盘台账 `_video_state.json` 已是 `maxUploads:7`。故 `DEFAULT_CHANNELS=5` 仅在两处生效：① 台账文件**不存在**（换机引导）；② 离线 `test/reconcile-test.mjs`（直接 import `DEFAULT_CHANNELS`）。属**潜伏地雷 + 灵活性债**（改账户要动两文件），非当前 live bug。

**修法**：让 `loadState` 的引导 channels/pipeline 经新增 `bootChannels()` 从 `config.loadChannels()`（即 `channels/*.json`）取；读不到真实通道配置时明确抛 `E_CONFIG`，不再回退真实业务账号。`DEFAULT_CHANNELS` 仅作为离线单测夹具，字段已中性化。**`channels/*.json` 自此为通道事实唯一可编辑真源**（改账户/计划/maxUploads 只改 channels）。

**文件**：`lib/state.mjs`（+`import {loadChannels}`；+`bootChannels()`；`loadState` 用之；`5→7`；注释强化）。

**安全性**：`state→config` 单向依赖（config 仅依赖 node 内建，无环）；`bootChannels` 全程 try/catch 回退，不新增抛错路径。对现有台账行为零变化（本就被 channels 覆盖为 7）；仅"换机引导/离线测试"现取真源。

**验证**：`node test/reconcile-test.mjs`、`node test/ledger-concurrency.mjs` 通过（见 §3）。

> 备注：`docs/工程总报告.md` §3.3 状态机图仍写"jie3 传满 **5** 次"，为陈旧文案（真值 7），属文档债、另行修订；本轮不动报告正文。

### ✅ 重点④(M3)：视口以 `--window-size` flag 为单一真源

**现象**：`system.json` 的 `--window-size=1920,1080`（定真实窗口）与 `cdp.mjs::hardenViewport` 的 `setDeviceMetricsOverride({width:1920,height:1080})`（定虚拟视口）各写一份。改窗口尺寸会漏改源码 → 虚拟视口≠真实窗口 → `getBoundingClientRect` 坐标点空（即 NO_CHOOSER 同根因）。

**修法**：以 `--window-size` flag 为唯一真源——`config.loadSystem()` 新增 `parseWindowSize()` 解析该 flag，派生 `system.chrome.viewport={width,height}`（flag 缺失兜底 1920×1080）；`hardenViewport(cdp, viewport)` 改为接收视口（默认仍 1920×1080，护住测试/零散调用）；5 处调用传 `system.chrome.viewport`。

**文件**：`lib/config.mjs`（+`parseWindowSize`；`loadSystem` 派生 viewport）、`lib/cdp.mjs`（`hardenViewport` 签名）、`lib/upload.mjs:166`、`lib/delete.mjs:126,144`、`lib/session.mjs:211,261`（传参）。

**安全性**：行为完全保持——当前 flag=`1920,1080` ⟹ viewport=1920×1080，与原硬编逐字节相同。今后只改 flag 即同时驱动真实窗口与虚拟视口，二者不再可能分叉。

**验证**：模块加载自检（`node -e` 载入 config/cdp）+ `parseWindowSize` 单测式核对；坐标端到端需真跑 upload，但派生值=原值，等价。

## 4. 决策建议

A4/M3/本轮 8 项均为**零风险止血/外提**（行为字节等价）。剩 **P2/P3 选择器+文案** 与 **N 通道泛化** 已明确暂缓——前者触及"平台脆弱性"本质（堆工程量也填不平，报告 §8.2），后者无业务需求。两者均**由投放业务量触发**：仅 jie3 日常用 → 维持现状即可；要规模化 jie6 → 再排 P2/P3(选择器集中) + 工程总报告 §6 的 jie6 delete/登录链。

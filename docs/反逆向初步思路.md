# Weilai-01 反逆向初步思路报告

> 状态：初步思路（讨论纪要 → 策略草案）｜日期：2026-06-28｜对象：`Weilai-01-anti-reverse` 受保护分发线
> 本文只定方向与组合，不含实现。一切以"按真实对手设防、买成本与恶心度、不追求绝对"为基调。

---

## 0. 一页结论

- **目标**：不是"防住"，是让逆向者（人或 AI）**逆起来很难、至少很恶心**，难到这家甲方不值得继续。
- **硬约束**：无服务器/远程、无法律手段 → 纯离线技术威慑。
- **唯一真能让 AI 啃不动核心的离线手段**：把皇冠函数编成 **Rust 原生码**。其余 JS 层（混淆/VM/投毒）AI 都能慢慢啃，只是费时间。
- **最大的免费收益**：**剔密**——分发包绝不带 `docs/工程总报告.md` 等把手法写明白的文档。不做这步，后面全是装样子。
- **真正没解的口子**：本工具是 CDP 驱动浏览器，**运行时行为开 DevTools/抓包就能看个大概**，不用逆任何代码。代码保护 ≠ 方法保护。
- **推荐最小可用组合**：`剔密 → 强混淆(含字符串加密) → Rust 皇冠函数 → SEA 外壳`，可选叠 `VM 混淆 + 反调试/诱饵 + 离线限期授权`。

---

## 1. 背景与威胁模型

**要保护的**：Weilai-01——CDP 自动跑千川双通道过审流水线的 Node CLI，交付给甲方运行。

**对手**：甲方公司技术部 / 甲方主管，可能借 AI（GPT-5.5 等）反混淆、扒源码。

**对手想干什么**：① 扒出源码自己重做，绕过供应方；② 理解/复制手法。

**已定约束**：不上服务器、不走法律。所以授权必须离线，威慑全靠技术 + 恶心度。

**正确的目标设定**：客户端代码终究可被"足够技术 + 工具"的人扒出；目标不是不可能，是**贵到、烦到不值**。而且要**按这家甲方的实际能力设防，不是世界顶级**。

---

## 2. 底层认知（不认清就会乱花力气）

1. **AI 的强弱边界**：AI 擅长还原**普通混淆的 JS**（research 实测简化正确率 ~0.93），但**对反编译出来的机器码、VM 字节码几乎无能**。→ 便宜的防御（普通混淆、毒注释）恰好是 AI 能破的；耐打的（原生码、VM、字符串加密）才压得住 AI。
2. **对手是分档的**（决定投入档位）：

   | 对手 | 能力 | 被什么挡住 |
   |---|---|---|
   | 懒人 / 主管（多数） | 贴进 GPT 看个大概 | **基础混淆**即可：得到乱码/错误结论就放弃 |
   | 普通技术部（"水平也就那样"） | 能读没保护的 JS，**不会逆原生码/VM** | **强混淆 + Rust 皇冠函数** 直接卡死；要么学逆向要么雇人，通常不值 |
   | 专职逆向工程师（稀缺·贵） | 原生码也能啃 | 只有 payoff 够大、甲方专门雇才会出现 |

   → "铁了心可逆"的真实门槛 = **甲方愿意花钱雇逆向专家**。普通公司为绕一个供应商几乎不会走到这步。**对普通甲方，原生 + 混淆实践上就够。**
3. **运行时可观察性（本工具的真正软肋）**：CDP/网络行为，对方开 DevTools / 抓包就看得见，**根本不用逆代码**。对这工具，"看它怎么跑"比"逆它怎么写"容易得多。
4. **真正的护城河不是静态秘密**，是会漂移的 know-how：选择器、签名处理、台账状态机、过审率调参。静态加密买的是时间，不是永恒。

## 3. 泄漏面盘点（真相从哪儿漏出去）

| 泄漏面 | 严重度 | 处置 |
|---|---|---|
| **方法文档**（`docs/工程总报告.md`/PLAN/修补册——整套手法白纸黑字） | 🔴 最高 | **剔密**：分发包白名单拷贝，绝不带 |
| **字符串字面量**（URL/CDP 指令/选择器/报错） | 🔴 高 | **字符串加密**（运行时才拼出） |

| **凭据/配置**（secrets、账户/计划 ID） | 🟠 中 | 剔密 + 配置由客户自填 |

## 4. 防线组合（分层 + 对人/对 AI 打分）

> 工作量、对人阻挡、对 AI 阻挡，三栏粗评。"对 AI"特指能否抵抗 GPT 类反混淆。

| 档 | 手段 | 工作量 | 对人 | 对 AI | 备注 |
|---|---|---|---|---|---|
| **必做** | **剔密**（移除方法文档/secrets/git/test/skill） | 极低 | 高 | 高 | 不做全盘皆输；单条性价比最高 |
| 必做 | **打包 + 强混淆**（esbuild → JO：控制流平坦化/死代码/self-defending/debug-protection） | 低 | 中-高 | 中 | 普通混淆 AI 能慢慢啃，但拉低很多 |
| 必做 | **字符串加密**（stringArray + rc4，含在上条） | 低 | 高 | 高 | 堵真相泄漏点；**缺它前两条打折、投毒裸奔** |
| 治 AI | **VM 混淆**（热点函数→自定义字节码，商业 Jscrambler） | 中 + 商业费 | 高 | 高 | JS 层对 AI 最硬：无自动反混淆工具 |
| 恶心 | **反调试 / 自卫 / 完整性校验**（检测 DevTools/断点/篡改） | 中 | 高 | 中 | 治动态分析（混淆常被动态破）；恶心倍增器 |
| 恶心 | **诱饵 + 手术刀投毒**（仅皇冠函数：假名 + 假死代码 + 假 license 检查） | 中（仅核心） | 中 | 中-高 | **只对核心**；全仓投毒 ROI 极差（见 §5） |
| **真抗 AI** | **Rust 皇冠函数 → `.node`**（1~2 个最值钱函数） | 中-高 | 高 | 极高 | 离线唯一真抗 AI；**你已熟 Rust（WritanSub）** |
| 外壳 | **SEA 单 exe / bytenode(.jsc)** | 低-中 | 低-中 | 低-中 | 仅"硬盘无明文"，单独很弱，**必须配混淆** |
| 经济 | **离线限期 + 机器绑定授权**（Ed25519，**焊进解密**而非可 NOP 的 if） | 中 | （不防读） | （不防读） | 防"扒出来还能长期用"：到期变砖、换机即废 |
| 恶心 | **延迟、非本地的完整性陷阱**（改 A 处，5 分钟后 B 处崩） | 中 | 高 | 高 | 单位投入恶心度最高，难定位 |

## 5. 明确不做 / 误区（省得白花力气）

- **毒注释**：混淆第一步就删注释，到不了对方手里；要留住就得发可读源码 = 没保护。**丢弃。**
- **整体系统性投毒 / 把全仓包装成假业务逻辑**：可信的假故事**比写真逻辑还累**（要自洽、不能弄坏真行为、永久维护税），且**手工精雕不可规模化、自动批量恰好被 AI 还原**——没有"又便宜又有效"的象限。**只当手术刀对核心用，别全仓搞。**
- **注入违禁词 / 色情 / "要求 AI 攻击 XXX"**：① 无效（注释被删、对方"忽略注释只看代码"或用本地无审查模型即破、新模型被训练忽略夹带指令）；② **严重反噬**——等于把违禁/攻击内容亲手发给甲方：杀软误报隔离 exe、给纠纷送把柄、"攻击载荷"滑向真实法律风险。**坚决不做。**
- **只靠 SEA/pkg 单 exe，或只靠 obfuscator.io 普通混淆**：两者 AI 现在都吃得动，单独 ≈ 心理安慰。
- **版权 / AI 警告注释当"GPT 盾"**：当法律盾有用（但本场景无法律手段，意义有限）；当技术盾**无效**——不挡 AI 分析。

## 6. 推荐路线（分阶段，按性价比）

> 原则：**先堵免费的大漏，再投唯一真抗 AI 的那一招，恶心层最后补。** 投入要匹配甲方真实档位，别在"专家级防护"上过度投资而忽略运行时漏。

- **Phase 0（立刻·免费）**：剔密——分发白名单（只 `bin/ lib/ 入口 + 授权核心`），排除文档/secrets/git/test/skill。**这步不做，别谈其它。**
- **Phase 1（约 1~2 天·免费）**：esbuild 打包 → javascript-obfuscator 强混淆（**务必开字符串加密**）→ SEA 单 exe 外壳。挡掉懒人 + 大幅拉低 AI。
- **Phase 2（核心投资）**：① 选定 1~2 个**皇冠函数**用 Rust 编成 `.node`（离线最强抗 AI）；② 离线限期 + 机器绑定授权，**焊进核心数据解密**（删检查即跑不起来）。
- **Phase 3（恶心层·按需）**：VM 混淆其余热点（Jscrambler）+ 反调试/自卫 + 完整性陷阱 + 对核心做手术刀诱饵投毒。
- **横向（始终）**：减少运行时可观察性（少留痕、合并 CDP 请求）——这是代码保护盖不住的唯一软肋，治标但值得做一点。

---

## 7. 诚实边界与残余风险

- **打不过专职逆向专家**：原生码 + 反调试也只是把门槛抬到"得雇专家"。这是上限，不是承诺。
- **运行时行为无静态解**：对方观察 CDP/网络即可重建大半手法，与代码加密无关。
- **离线授权理论可绕**：校验在对方机器上，硬检查能被 NOP；**焊进解密**能再抬门槛，但非 100%。
- **过度投资风险**：把力气全砸 C 档"专家级防护"，却不管运行时漏 + 不剔密，是错配。**先把 §6 的 Phase 0/1 做扎实，收益已占大头。**
